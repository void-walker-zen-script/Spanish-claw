# Spanish Claw 任务清单

## 主要差异与合并原则

你提供的版本更适合作为当前 MVP 执行主线，因为它已经明确采用 Flask、Jinja templates、static 静态资源和本地 JSON 数据；上一版更偏通用 Web 产品拆解，包含 React、TypeScript、组件目录等假设。

本版已做如下优化：

- 去除重复项：把“项目对齐”和“文档搭建”合并为文档与范围阶段。
- 技术路线收敛：删除 React、TypeScript、src 目录等不符合当前 Flask MVP 的内容。
- 流程前后顺序优化：先定 demo 素材，再建数据，再做路由和页面交互。
- 保留必要体验任务：补回空状态、未标注词默认状态、移动端布局、收藏删除与筛选、手动验证。
- 明确 Demo 路径：让任务最终服务于一次可演示的完整学习闭环。

## 最适合展示的 Demo 顺序

1. 打开首页，说明产品定位。
2. 进入 lesson 页面。
3. 播放音频或展示听力材料。
4. 点击一个单词，展示词义、原形、变位和例句。
5. 点击一句句子，展示句子层解释和地道表达。
6. 点击收藏。
7. 进入我的收藏页，展示沉淀能力。
8. 最后说明未来如何扩展到社区、AI 解释和 App。
