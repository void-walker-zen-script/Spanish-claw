# Spanish Claw Web MVP 设计文档

## 1. 项目名称
Spanish Claw Web MVP

## 2. 产品定位
Spanish Claw 是一个面向西班牙语学习者的交互式听力与发音学习网页产品。

它不是传统意义上的单点词典，也不是普通听力播放器，而是一个把“听力输入、词句点按查询、表达延展、发音练习和学习沉淀”整合在一起的轻量学习工作流产品。

首版目标不是做完整语言平台，而是验证一个核心命题：

**当用户在听西语材料时，如果可以直接点击单词或句子并获得多维解释，并能在发音练习中获得可见的口型反馈，学习体验是否会明显优于“听力 App + 词典 App + 发音工具来回切换”的方式。**

---

## 3. 产品目标

### 3.1 首版目标
首版 Web MVP 主要验证以下几点：

1. 用户是否会对“边听边点边查”的交互产生明显兴趣。
2. 用户是否会认为多维解释卡片比普通查词更有价值。
3. 用户是否愿意收藏词句，形成个人学习沉淀。
4. 用户是否会认为发音口型反馈比单纯录音回放更直观。
5. 这一套交互是否足够清晰，适合进一步扩展为完整产品或 App。

### 3.2 非首版目标
第一版暂不追求：

- 海量内容库
- 真正完整的词典系统
- 用户评论与弹幕社区
- 审核后台
- 多端同步
- 会员体系
- 真正商业化闭环
- 高精度发音评分模型
- 完整口语课程体系

---

## 4. 核心用户路径
首版产品的核心路径如下：

### 路径 A：从首页进入学习页
用户进入首页
-> 看到产品名称、简介和进入按钮
-> 点击“开始学习”
-> 进入 lesson 页面

### 路径 B：在听力页进行点击学习
用户进入 lesson 页面
-> 看到西语材料与音频
-> 点击某个单词
-> 右侧展示解释卡片
-> 继续点击其他词或句子
-> 对比不同解释与表达方式

### 路径 C：收藏重点内容
用户在解释卡片中发现重要词句
-> 点击收藏
-> 系统保存到本地收藏列表
-> 用户进入 saved 页面查看已收藏内容

### 路径 D：进行发音练习
用户选择某个音素、单词或句子
-> 打开摄像头
-> 按照目标进行跟读或口型练习
-> 系统基于口型拟合给出相似概率
-> 系统输出发音提示和鼓励语句

这一条路径构成首版最重要的产品闭环：

**进入材料 -> 点击理解 -> 获得解释 -> 进行收藏 -> 形成沉淀**

发音练习路径形成补充闭环：

**选择目标 -> 打开摄像头 -> 练习口型 -> 获得反馈 -> 继续练习**

## 5. 核心模块设计

### 5.1 首页模块

#### 功能
向用户说明产品是什么，并提供进入学习页的入口。

#### 页面内容
- 产品名称
- 一句话定位
- 简短产品介绍
- “开始学习”按钮

#### 作用
首页主要承担“第一印象”和“进入主流程”的作用，不承担复杂交互。

---

### 5.2 听力学习页模块

#### 功能
展示一篇西语材料及其音频，并承载核心点击交互。

#### 页面内容
- 材料标题
- 音频播放器
- 西语文本区域
- 解释卡片区域
- 收藏按钮区域

#### 作用
这是整个 MVP 的核心页面，也是最主要的演示页面。

---

### 5.3 单词点击模块

#### 功能
允许用户点击文本中的特定单词，并查看对应词条信息。

#### 输出内容
单词解释卡片应至少包含：

- 中文直译
- 词性
- 原形
- 变位说明
- 同义词
- 反义词
- 一般表达
- 地道表达
- 例句

#### 作用
这是产品最核心的差异化能力之一。

---

### 5.4 句子点击模块

#### 功能
允许用户点击整句或特定句段，查看更高层次的句子理解信息。

#### 输出内容
句子卡片可包含：

- 句子直译
- 句子意译
- 一般表达方式
- 更自然或更地道的表达方式
- 使用场景说明

#### 作用
相比只查单词，句子点击模块更能体现产品对“语境理解”的重视。

---

### 5.5 解释卡片模块

#### 功能
统一承载所有查询结果。

#### 设计要求
- 位置固定在页面右侧，便于持续查看
- 点击不同词句时，右侧内容即时更新
- 字段标签明确，层次清晰
- 适合阅读和演示

#### 作用
解释卡片是整个交互价值的集中体现，也是用户感知“这个产品不只是普通词典”的关键。

---

### 5.6 收藏模块

#### 功能
用户可将当前词条或句子加入个人收藏。

#### MVP 设计
- 使用浏览器本地存储或假数据实现
- 收藏成功后给出明确反馈
- 提供我的收藏页进行查看

#### 作用
把一次性点击学习变成可积累的长期学习行为。

---

### 5.7 我的收藏模块

#### 功能
展示用户已收藏的词和句子。

#### MVP 内容
- 收藏单词列表
- 收藏句子列表
- 每项可展示简要释义或标签

#### 作用
体现产品的“学习沉淀”价值，为未来扩展学习档案和社区打基础。

---

### 5.8 Pronunciation Coach（纠音助手）

#### 功能
- 用户打开摄像头练习某个音素、单词或句子
- 系统基于口型拟合给出相似概率
- 系统输出鼓励式反馈

#### 输入
- 用户摄像头画面
- 当前练习目标：音素 / 单词 / 句子

#### 输出
- 相似概率
- 发音提示
- 鼓励语句

#### MVP 设计
- 首版可以先做成独立练习区域或 lesson 页面中的轻量入口。
- 练习目标先使用本地 JSON 假数据配置。
- 相似概率可先使用模拟分数或基础口型检测结果，用于验证交互闭环。
- 反馈文案采用鼓励式表达，避免让用户产生挫败感。

#### 作用
纠音助手把被动模仿变成可见反馈，帮助用户从“听懂和看懂”继续走到“说出来并持续修正”，提高输出练习的连续性和可坚持性。

## 6. 产品结构设计

### 6.1 页面结构
首版产品建议包含以下页面：

#### index.html
首页

#### lesson.html
听力学习页

#### saved.html
我的收藏页

#### pronunciation.html
发音练习页或纠音助手页面

### 6.2 lesson 页面布局建议
推荐采用双栏布局：

#### 左侧区域
- 标题
- 音频播放器
- 西语文本

#### 右侧区域
- 解释卡片
- 收藏按钮
- 提示信息

这样可以让用户一边阅读文本，一边查看解释，而不需要频繁弹窗或跳转。

### 6.3 pronunciation 页面布局建议
推荐采用练习面板布局：

#### 左侧区域
- 当前练习目标
- 摄像头画面
- 开始 / 停止练习按钮

#### 右侧区域
- 相似概率
- 发音提示
- 鼓励语句
- 下一项练习入口

## 7. 数据设计

### 7.1 lessons.json
用于存储课程或材料数据。

建议字段：

- id
- title
- audio_url
- sentences

其中 `sentences` 可包含：

- sentence_id
- text
- tokens

### 7.2 vocab.json
用于存储词条和句子解释数据。

建议字段：

- key
- type：`word` / `sentence`
- translation
- pos
- lemma
- conjugation
- synonyms
- antonyms
- common_usage
- native_usage
- example

### 7.3 收藏数据
首版可使用：

- 浏览器 localStorage
- 本地假数据文件

首版不需要数据库。

### 7.4 pronunciation_targets.json
用于存储发音练习目标。

建议字段：

- id
- type：`phoneme` / `word` / `sentence`
- target
- display_text
- hint
- encouragements
- reference_note

## 8. 技术方案

### 8.1 前端
建议使用：

- HTML
- CSS
- JavaScript
- Web Camera API

原因：

- 学习成本低
- 开发速度快
- 适合 MVP
- 便于后续快速调整页面交互
- 可以先用浏览器摄像头能力验证口型反馈流程

### 8.2 后端
建议使用：

- Python
- Flask

原因：

- 结构轻量
- 对当前阶段友好
- 适合快速做本地演示
- 便于以后接入 API 或数据处理逻辑

### 8.3 数据来源
首版使用本地 JSON 假数据。

原因：

- 可控
- 开发快
- 不依赖外部接口
- 更适合先验证交互体验

### 8.4 后续扩展空间
当 MVP 验证有效后，可以继续扩展为：

- API 查询真实词典数据
- 接入口型关键点检测或机器学习模型
- 更真实的发音评分与音频分析
- 用户系统
- 收藏云同步
- 评论和用户提交
- 审核机制
- App 化
